# Quant Ecosystem Brand Guidelines

## Mission and Vision

The Quant Ecosystem is a unified suite of productivity and communication applications designed to empower individuals and teams across India and beyond. Our brand reflects innovation, accessibility, and cultural sensitivity.

**Mission**: Build world-class productivity tools that respect linguistic diversity and accessibility needs.

**Vision**: A unified digital workspace where every app feels like part of a cohesive family, regardless of language or ability.

## Brand Values

- **Unity**: All apps share a common design language
- **Accessibility**: WCAG AA compliance minimum, AAA for high-contrast mode
- **Cultural Sensitivity**: First-class Indic script support
- **Performance**: Lightweight tokens and assets
- **Consistency**: Predictable patterns across all touchpoints

## Logo Usage

### Primary Logo

The Quant wordmark and symbol are the primary brand identifiers. They are available in light and dark variants.

### Clear Space

Maintain a minimum clear space around the logo equal to the height of the "Q" in the wordmark. This ensures the logo remains legible and uncluttered.

### Minimum Size

- Wordmark: minimum width of 120px on screen, 30mm in print
- Symbol: minimum width of 24px on screen, 8mm in print

### Logo Don'ts

- Do not rotate the logo
- Do not stretch or distort
- Do not apply drop shadows or effects
- Do not place on busy backgrounds without sufficient contrast
- Do not modify colors outside the approved palette
- Do not add outlines or borders
- Do not rearrange logo elements

## Iconography

### Design Principles

All icons in the Quant ecosystem follow these principles:

1. **24x24 viewBox**: Standard canvas size for all icons
2. **2px stroke**: Consistent weight across the set
3. **Round caps and joins**: Friendly, approachable aesthetic
4. **No fill**: Outline-only style using currentColor
5. **Geometric simplicity**: Clean, recognizable silhouettes

### Icon Categories

- **Navigation**: menu, arrows, chevrons, home, back
- **Actions**: search, filter, sort, refresh, download, upload, share, copy, paste, cut, delete, edit, save, undo, redo, plus, minus, close, check
- **Communication**: bell, inbox, send, attach, link, unlink
- **Media**: play, pause, stop, volume controls, mic, camera, image, film, music
- **Status**: info, warning, error, success, help, loading, clock
- **Interface**: settings, user, lock, eye, star, heart, bookmark, pin, flag, tag, grid, list, maximize, minimize, external-link, more
- **Files**: file, folder, document, code, archive, cloud, database
- **Devices**: phone, tablet, laptop, monitor, printer, wifi, bluetooth, battery

### Per-App Icons

Each of the 16 apps has a dedicated icon used in navigation, app switchers, and branding contexts. App icons use the app's brand color as the stroke color.

## Motion

### Spring Physics

Animations use spring-based motion for natural, responsive feel:

- **Default Spring**: stiffness 300, damping 30, mass 1
- **Gentle Spring**: stiffness 150, damping 20, mass 1
- **Bouncy Spring**: stiffness 400, damping 15, mass 1
- **Stiff Spring**: stiffness 500, damping 40, mass 1

### Easing Curves

- **Ease Out**: cubic-bezier(0.16, 1, 0.3, 1) - elements entering
- **Ease In**: cubic-bezier(0.55, 0.055, 0.675, 0.19) - elements exiting
- **Ease In Out**: cubic-bezier(0.87, 0, 0.13, 1) - elements transitioning
- **Decelerate**: cubic-bezier(0, 0, 0.2, 1) - elements settling
- **Linear**: linear - progress indicators

### Duration Scale

- **Instant**: 50ms - micro-interactions, state changes
- **Fast**: 150ms - hover states, small transitions
- **Normal**: 250ms - standard transitions
- **Moderate**: 350ms - medium complexity animations
- **Slow**: 500ms - page transitions, large elements
- **Glacial**: 800ms - dramatic reveals, onboarding

## Themes

The Quant ecosystem supports 6 curated themes:

### Dark (Default)

Deep dark surface with indigo primary and amber accent. Designed for prolonged use in low-light environments.

### Light

Clean white surface with indigo primary. Optimized for readability in well-lit environments.

### Neon

High-energy dark theme with vivid green primary and magenta accent. For creative and entertainment contexts.

### Bharat

Warm, culturally-inspired palette with saffron primary and deep purple accent. Celebrates Indian aesthetic traditions.

### High Contrast

Maximum contrast (21:1 ratio) with pure black background and white foreground. Yellow primary for clear focus indication. Meets WCAG AAA requirements.

### Colorblind Safe

Carefully selected colors that remain distinguishable for common forms of color vision deficiency (deuteranopia, protanopia, tritanopia). Uses blue/orange safe palette.

## Accessibility Requirements

### Contrast Ratios

- **Normal text (AA)**: Minimum 4.5:1 contrast ratio
- **Large text (AA)**: Minimum 3:1 contrast ratio
- **Normal text (AAA)**: Minimum 7:1 contrast ratio (highContrast theme)
- **UI components**: Minimum 3:1 against adjacent colors

### Testing

Use the built-in contrast utilities to validate color pairs:

```typescript
import { meetsAA, meetsAAA, contrastRatio } from '@quant/brand';

// Check if a color pair meets WCAG AA
meetsAA('#0F172A', '#FFFFFF'); // true

// Get the exact contrast ratio
contrastRatio('#000000', '#FFFFFF'); // ~21
```

### Indic Script Support

All text rendering must account for:

- Larger line heights for Indic scripts (complex glyph shaping)
- Proper font fallback chains
- Bidirectional text support where needed
- Testing with actual Indic text, not just Latin characters

## CSS Custom Properties

### Brand Variables

Generate brand CSS variables using:

```typescript
import { generateBrandCSS } from '@quant/brand';
const css = generateBrandCSS();
```

This produces a complete `:root` block with all color, typography, and motion variables.

### Theme Variables

Generate theme-specific CSS:

```typescript
import { generateThemeCSS } from '@quant/brand';
const css = generateThemeCSS('dark');
```

This produces a `[data-theme="dark"]` selector block with semantic color variables.

### App Variables

Generate per-app CSS:

```typescript
import { generateAppCSS } from '@quant/brand';
const css = generateAppCSS('quantmail');
```

## Implementation Notes

- All brand tokens are exported as `const` objects for tree-shaking
- SVG logos and icons are stored as template literal strings
- The package is ESM-only (`"type": "module"`)
- TypeScript strict mode is enforced
- No runtime dependencies - pure token definitions
