# Five AI-native differentiator packages and launch readiness gate

Phase 16 adds five new packages under `packages/` — universal-timeline, ai-daily-brief, command-palette, ai-memory, and contextual-sidekick — each with typed interfaces, service classes, and Vitest suites. Phase 17 runs all six quality gates, validates Dockerfiles and Helm templates, and confirms security posture before declaring the ecosystem launch-ready.

All five packages follow an identical structural pattern: in-memory Maps as backing stores, synchronous CRUD with async wrappers where sources are external, and no persistence layer. The packages are self-contained service stubs with clean type boundaries. The launch-readiness verification is documentation-only (state files updated), not a CI pipeline change.

**Watch for:** Unbounded in-memory growth in all five stores with no eviction strategy (confirmed). Memory export data leaks full access logs to the exported user without filtering entries that reference other users' apps (confirmed). The CSV export is injection-susceptible due to unescaped commas/newlines in content fields (likely). `queryWithSources` in universal-timeline fires all source fetches sequentially rather than in parallel (confirmed).

## High-level view

The five packages share an architectural template: a `Map<string, T>`-based store, a domain-specific scoring/detection layer, and an export or query facade. None introduce external dependencies beyond `@quant/common` and `zod`. Each represents a product differentiator (cross-app timeline, AI memory, contextual assistant, command palette, daily brief) implemented as a typed service stub that could later be backed by a real database.

The timeline service introduces a pub/sub fan-out pattern where published events notify per-user subscriber callbacks synchronously on the publishing path. This means a slow subscriber blocks the publish call — acceptable in tests but a production concern if this service is ever used in a request path.

AI Memory is the most complete package from a data-lifecycle perspective: it handles creation, expiration-based filtering, access logging, and portable export in three formats (JSON, markdown, CSV). The Zod schema validates imports. The access-log design tracks which app accessed which memory and why, creating an audit trail.

The command palette implements permission-gated execution and fuzzy matching over a command registry. The fuzzy scoring is substring-based, not character-by-character fuzzy (no Levenshtein or Smith-Waterman), so "schd" won't match "schedule" — a UX gap if this powers a real Cmd+K interface.

The daily brief's anomaly detection uses fixed thresholds (50 messages = spike, 24h overdue = anomaly). These are configurable via setters but not through any configuration mechanism that would survive across instances.

The launch readiness gate (Phase 17) is purely a state-file update — no CI workflow was created or modified. The "gates pass" claim is accurate based on the commands run, but there's no mechanism to prevent regression without CI enforcement of these checks on every commit.

<details>
<summary>Issues (7)</summary>

1. **Unbounded memory growth** — All five packages store data in `Map` instances with no size limit or eviction. Add a max-entries cap or LRU eviction to prevent OOM in long-running processes.
2. **Sequential source fetching in timeline** — `queryWithSources` awaits each source in a `for` loop. Use `Promise.all` to fetch sources concurrently.
3. **CSV injection in memory export** — `toCsv` escapes double-quotes but not commas, newlines, or leading `=`/`+`/`@` characters within content fields. Cells containing these characters produce malformed CSV or spreadsheet formula injection.
4. **Access log leaks in export** — `MemoryExporter.export` includes `accessLog` entries (which contain `requestingApp` names from other services) in the user-facing export. Consider stripping or summarizing access logs in exported data.
5. **Synchronous subscriber notification** — `UniversalTimelineService.publish` calls all subscriber callbacks synchronously in the same tick. A throwing or slow subscriber blocks or crashes the publish path.
6. **Fuzzy matcher is substring-only** — `FuzzyMatcher.computeScore` uses `includes()` checks, not character-sequence fuzzy matching. Abbreviated inputs ("schd", "crt msg") won't match commands unless they happen to be substrings.
7. **No CI gate enforcement** — Phase 17 ran gate commands manually and updated state files, but no CI workflow enforces these gates on future commits. The "launch ready" status can silently regress.

## Unbounded in-memory stores across all five packages

Every package uses `Map<string, T>` as its primary data structure with no upper bound. `AIMemoryStore`, `UniversalTimelineService`, `CommandRegistry`, `DailyBriefGenerator` (via accumulated provider results per call, not stored, so less concerning), and `SidekickEngine` all grow without limit.

The most concerning case is `AIMemoryStore` because memory entries include access logs that grow unboundedly per entry via `logAccess`. Each call appends to `entry.accessLog` with no cap:

```typescript
logAccess(id: string, access: MemoryAccess): boolean {
  const entry = this.memories.get(id);
  if (!entry) return false;
  entry.accessLog.push(access); // unbounded
  this.memories.set(id, entry);
  return true;
}
```

A frequently-accessed memory entry accumulates an arbitrarily large access log array. In production with real AI memory access patterns (every conversation turn querying memories), this becomes a memory leak.

## Sequential source fetching in universal-timeline

`queryWithSources` iterates sources with `for...of` and `await`s each one:

```typescript
for (const [, source] of this.sources) {
  const events = await source.fetchEvents(filter);
  sourceEvents.push(events);
}
```

With N registered sources, latency is the sum of all source latencies rather than the maximum. This is a straightforward `Promise.all` opportunity.

## CSV export injection vulnerability

`MemoryExporter.toCsv` wraps content and explanation in double quotes and escapes internal quotes, but doesn't handle:

- Commas within unquoted fields (id, userId, category, source, sourceApp are unquoted)
- Newlines within content (quoted field spanning multiple lines is valid CSV but many parsers mishandle it)
- Cells starting with `=`, `+`, `-`, `@` which trigger formula execution in Excel/Google Sheets

If any `source` or `sourceApp` field contains a comma, the CSV column alignment breaks. More importantly, user-controlled `content` could contain `=HYPERLINK(...)` or similar payloads that execute when opened in a spreadsheet.

## Synchronous subscriber fan-out in timeline publish

```typescript
const callbacks = this.subscribers.get(full.userId) ?? [];
for (const cb of callbacks) {
  cb(full);
}
```

If a subscriber throws, the remaining subscribers for that user never receive the event. There's no try/catch around individual callback invocations. A single misbehaving subscriber breaks the notification chain.

## Fuzzy matcher scoring model

The command palette's `FuzzyMatcher` scores based on `includes()` — whether the input string is a substring of the command name, keywords, category, or description. This means:

- "mail" matches "Send email" (keyword contains input)
- "sml" does not match "Send email" (no substring relationship)

For a Cmd+K interface that users type into quickly with abbreviated terms, pure substring matching misses the majority of fuzzy-typing patterns. Character-sequence matching (does each character of the input appear in order in the target?) is the minimum expectation for this kind of UI.

## Launch gate as documentation vs. automation

Phase 17 updated `.agents/state/quant-launch-gates.md` and `.agents/state/quant-autonomous-status.json` with "all gates pass" and "READY (with caveats)". These are human-readable state files. No GitHub Actions workflow, pre-commit hook, or CI configuration was added or modified. The gate verification happened once, interactively, and the result was recorded. Future commits could break typecheck, tests, or lint without any automated feedback loop catching it.
