# Twilio SIP phone provider with call session management (v2)

This change adds a dual-mode Twilio phone provider, call session state machine, TwiML builder, native phone fallback, and CallAgent for Quant Live voice intents. The v2 revision addresses all five issues from the prior review: live mode now instantiates the real Twilio client via a factory function, TwiML escapes XML special characters, CallSessionManager has `removeStateChangeListener`, NativePhoneProvider has dedicated tests, and comments document the permission tier design and credential handling.

**Watch for:** The `createTwilioClient` factory uses `require('twilio')` in an ESM package (`"type": "module"`, `"module": "ESNext"` in tsconfig), which will throw at runtime in Node.js ESM mode unless bundled or transpiled first (confirmed).

## High-level view

The live-mode fix correctly instantiates the Twilio SDK via `createTwilioClient(accountSid, authToken)` when no client override is injected. The factory uses `require('twilio')` with an eslint-disable comment — this works under vitest (which uses a CJS-compatible module system) but will fail at runtime if the compiled output runs as native ESM in Node.js. Since this package targets `"module": "ESNext"` and `"type": "module"`, a dynamic `await import('twilio')` or a top-level import would be the safe choice.

## `require()` in an ESM context

The `createTwilioClient` factory at line 16 of `twilio-phone-provider.ts`:

```typescript
function createTwilioClient(accountSid: string, authToken: string): TwilioClientLike {
  // eslint-disable-next-line @typescript-eslint/no-require-imports
  const Twilio = require('twilio') as (sid: string, token: string) => TwilioClientLike;
  return Twilio(accountSid, authToken);
}
```

The package declares `"type": "module"` in `package.json` and compiles with `"module": "ESNext"`. In Node.js 22+ (the stated minimum), `require()` is not available in ESM contexts unless behind `--experimental-require-module` or wrapped in `createRequire`. The eslint-disable comment silences the lint warning but doesn't address the runtime incompatibility.

This doesn't surface in tests because vitest transforms modules with its own loader. It will surface when the compiled `.js` output is imported by any ESM entry point at runtime.

The simplest fix is to make the factory async and use dynamic import:

```typescript
async function createTwilioClient(
  accountSid: string,
  authToken: string,
): Promise<TwilioClientLike> {
  const { default: Twilio } = await import('twilio');
  return Twilio(accountSid, authToken);
}
```

This requires making the constructor defer client creation to first use (lazy initialization), or moving client creation into `initialize()`.
